Quick Start Guide

Get Started in 5 Minutes

This quick start guide will help you get up and running with the Admin Portal. Use it to manage employees, supervisors, cost centers, and system settings.

Logging In

1 Navigate to the Portal

Open your web browser and go to the Oxford House Expense Tracker web portal: https://oxford-house-mileage-tracker.vercel.app/. The same login screen is used for Staff, Supervisor, Senior Staff, and Admin.

2 Enter Credentials

Enter your admin email and password. After login, you will open the portal that matches your assigned access. If your account is assigned access to multiple portal types, you may see the portal switcher dropdown—if you think you should have it and don’t, reach out to your administrator(s).

Admin Portal Home

The Admin Portal opens to Employee Management with the Individual Management view. Here you see the current employees table, search, and buttons for View Archived, Add Employee, and Sync from HR API. Use the main tabs to switch to Supervisor Management, Cost Center Management, Reports & Analytics, Travel Reasons, Daily Description, or System Settings.

Adding Your First Employee

1 Click Add Employee

On the Employee Management tab, click the blue Add Employee button.

2 Fill In Details

Enter the employee's name, email, position, login role, phone, supervisor, and cost centers. Save the form.

3 Edit Anytime

Use the pencil icon in the Actions column to edit an employee, or the lock icon for access control. Use "View Archived" to see or restore archived employees.

Comprehensive Reference

Portal Tabs Overview

The Admin Portal has seven main tabs in the top navigation:

  • Employee Management – Manage employees (Individual Management and Mass Operations)
  • Supervisor Management – Assign and manage supervisors
  • Cost Center Management – Manage cost centers
  • Reports & Analytics – Run reports and view analytics
  • Travel Reasons – Configure travel reason options
  • Daily Description – Configure daily description options
  • System Settings – System-wide settings

Employee Management

Under Employee Management, the Individual Management sub-tab shows the current employees table. You can search by name, email, position, or phone; use View Archived to see or restore archived employees; Add Employee to create a new employee; and Sync from HR API to pull updates from your HR system. The table columns include Name, Email, Position, Login Role, Last Login, Phone, Supervisor, Cost Centers, and Actions (edit, lock). Use the Mass Operations sub-tab for bulk actions.

Sync from HR API

The Sync from HR API button (cloud icon next to Add Employee) pulls employee data from your configured HR API. Use it to keep the portal in sync with your HR system. Ensure the HR integration is configured in System Settings before syncing.

Review Changes Prompt

After syncing, you may see a Review Changes prompt that summarizes what will be added or updated (for example: new employees, title/position updates, supervisor assignments, or cost center changes). Review the list carefully, then confirm to apply the updates. If something looks off, cancel and double-check your HR integration settings before running sync again.

Supervisor Management

Use the Supervisor Management tab to assign employees to supervisors and manage the supervisor hierarchy. This determines who can approve expense reports for whom.

Cost Center Management

The Cost Center Management tab lets you create and manage cost centers. Employees and expenses are assigned to cost centers for reporting and approval workflows.

Edit Cost Center

Use the pencil/edit action on a cost center row to open the Edit Cost Center prompt. This is where you can update:

  • Cost Center Code - Short identifier used by the system and reports (leave empty when creating a new cost center to auto-generate from the name, if your system supports it).
  • Cost Center Name - The human-readable name shown throughout the portals.
  • Description - Optional notes to help admins understand what the cost center is for.
  • Enable Google Maps in Reports - When enabled, the Finance team can view route maps in PDF reports for this cost center (if map reporting is enabled and the report contains mileage routes).

Click Update to save changes, or Cancel to discard.

Per Diem Rules (Edit Rules)

Each cost center can have Per Diem Rules. Click Edit Rules in the Per Diem Rules column to configure how per diem eligibility is calculated for employees using that cost center.

Edit Per Diem Rules Prompt

The Per Diem Rules prompt lets you control how the system determines eligibility and payout. Common fields include:

  • Max Per Diem Amount - Maximum per diem amount per day.
  • Minimum Hours Worked - Minimum hours required to qualify.
  • Minimum Miles Driven - Minimum miles required to qualify.
  • Minimum Distance from Base - Minimum distance from the employee’s base address required to qualify.
  • Description - Optional description of the rule set.
  • Per Diem Type - Choose Fixed Amount (pays a set amount when eligible) or Actual Amount (pays actual per diem up to the max, depending on your configuration).

Click Save Rules to apply. These changes affect future per diem calculations for employees using this cost center.

Reports & Analytics

The Reports & Analytics tab provides reporting and analytics tools. Use it to run reports, view submission and approval metrics, and export data as needed.

Travel Reasons

Under Travel Reasons, you can define the list of travel reasons (or trip purposes) that staff can select when entering mileage or expenses. Keep the list aligned with your organization’s policies.

Daily Description Options

The Daily Description tab lets you manage the options or templates available for daily descriptions that staff use in their reports. This is separate from the staff-facing Daily Descriptions tab in their own reports.

System Settings

Use System Settings to configure system-wide options, including HR API integration, notification defaults, and other administrative preferences. Changes here apply across the portal.

Notifications

The notification bell in the top-right corner shows alerts such as new report submissions, approval requests, and system messages. Click the bell to open the notifications panel and review or clear items.

Tips & Best Practices

Keeping Data in Sync

Run Sync from HR API regularly (e.g. after HR updates) so employee and supervisor data stays current. Assign supervisors and cost centers to new employees as soon as they are added so they can submit reports correctly.

Supervisor and Cost Center Setup

Before staff submit reports, ensure every employee has a supervisor (in Supervisor Management or in the employee form) and at least one cost center. Use Cost Center Management to create or adjust cost centers that match your organization’s structure.

Tip

Use the search and filters on the Employee Management table to find employees quickly. Use "View Archived" when you need to restore or reference former employees.

Troubleshooting

Employee or List Issues

Employee Not Appearing

Problem: A new or updated employee doesn’t show in the list.

Solutions: Refresh the page; run Sync from HR API if you use HR integration; check "View Archived" in case the employee was archived. Ensure you’re on the Individual Management sub-tab.

Can’t Edit Employee

Problem: Edit (pencil) or other actions don’t work.

Solutions: Confirm you’re logged in as an admin; refresh and try again; check the browser console for errors. If the user is an admin, some fields may be restricted.

Sync from HR API

Sync Fails or No Data

Problem: Sync from HR API errors or doesn’t import anyone.

Solutions: Verify HR API is configured in System Settings; check credentials and endpoint URL; ensure the HR system is reachable from your network. Review any on-screen error message for details.

General Issues

Portal Slow or Unresponsive

Solutions: Refresh the page; try a different browser; clear cache. With large employee lists, the table may take a moment to load.

Missing Tabs or Access

Problem: You don’t see all Admin tabs.

Solutions: Confirm your login role is Admin. If you only have Supervisor or Staff access, you won’t see the full Admin Portal. Contact your system administrator to adjust roles.

Still Having Issues?

If problems persist, contact your system administrator or technical support with a description of the issue, steps you’ve tried, and any error messages or screenshots.

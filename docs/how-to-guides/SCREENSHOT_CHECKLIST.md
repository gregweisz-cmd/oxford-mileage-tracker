## Screenshot Capture Guidelines

### Mobile App Screenshots
- **Resolution**: Minimum 1080x1920 (portrait) or 1920x1080 (landscape)
- **Format**: PNG with transparency if needed, or JPG
- **Device**: Use actual device screenshots or high-quality emulator screenshots
- **Content**: Show actual app interface with sample data (no sensitive information)
- **Tips**: 
  - Use consistent device/emulator settings
  - Hide or blur sensitive personal information
  - Show key UI elements clearly
  - Capture full screen or relevant sections

### Web Portal Screenshots
- **Resolution**: Minimum 1920x1080
- **Format**: PNG preferred
- **Browser**: Use Chrome or Firefox at 100% zoom
- **Content**: Show actual portal interface with sample data
- **Tips**:
  - Use consistent browser window size
  - Hide sensitive data or use test accounts
  - Ensure text is readable
  - Capture full viewport or relevant sections
  - Use consistent theme/color scheme

### General Guidelines
- Use descriptive filenames (lowercase with hyphens)
- Keep file sizes reasonable (< 500KB per image)
- Add annotations if needed (arrows, highlights) using image editing software
- Ensure screenshots match the current UI (after recent updates)
- Test that screenshots display correctly in PDFs

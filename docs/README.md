# Documentation Structure

This directory contains all project documentation organized by category.

## 📁 Directory Structure

### `/user-guides/`
User-facing documentation for end users (staff, supervisors, finance team).

### `/admin-guides/`
Administrative documentation for system administrators.

### `/developer/`
Technical documentation for developers:
- Code architecture
- Development setup
- Testing guides
- API documentation

### `/deployment/`
Deployment guides and production setup instructions.

### `/api/`
API reference documentation.

### `/google-oauth/`
Google OAuth implementation guides and setup instructions.

### `/archive/`
Archived documentation from past sessions and troubleshooting:
- `/troubleshooting/` - Resolved issues and debugging guides
- `/deployment/` - Historical deployment guides
- `/scripts/` - One-time deployment scripts
- `/sessions/` - Past session summaries

## 📝 Documentation Guidelines

When creating new documentation:

1. **User Guides** → `/user-guides/`
2. **Admin Guides** → `/admin-guides/`
3. **Developer Docs** → `/developer/`
4. **Deployment** → `/deployment/`
5. **Troubleshooting** → `/archive/troubleshooting/` (after resolved)

Keep active, current documentation in the root directory. Archive old or resolved items.
